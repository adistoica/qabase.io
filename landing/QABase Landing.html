FAQs

What is QABase?
+
QABase is a test management and intelligence platform. It unifies manual test cases, automated runs from your CI, and AI‑assisted authoring into a single workspace — so your team has one place to plan, track, and decide whether a release is ready.
How is QABase different from other QA tools?
+
Most QA tools either store manual test cases or aggregate CI logs — rarely both. QABase is built around the join between them, plus AI helpers that convert cases into framework‑native scripts, and a release‑readiness score grounded in actual signal, not a green‑light heuristic.
What is AI Drafts, and how does AI work in QABase?
+
AI Drafts converts plain‑English test cases into Playwright, Cypress, Selenium, or Detox scripts you can review before merging. AI never auto‑mutes a test or hides a failure. It assists; engineers stay in the loop.
Which integrations does QABase support?
+
CI/CD: GitHub Actions, GitLab CI, CircleCI, Jenkins, Buildkite. Frameworks: Playwright, Cypress, Selenium, Detox, Pytest, JUnit. Issue trackers: Jira, Linear, GitHub Issues, ClickUp. Plus webhook + REST API for everything else.
What is the QABase MCP server?
+
An open‑source Model Context Protocol server that lets coding assistants read and write to your QABase workspace — generate cases, run suites, summarize failures — from inside Cursor, Claude Code, and any other MCP‑aware client.
How is QABase priced?
+
Free for teams up to 5 users with unlimited projects and cases. Team plans start at $19/seat/month, billed annually. Enterprise plans include SSO, SCIM, audit logs, and SOC 2 reports. See the pricing page for details.
How does QABase help teams make release decisions faster?
+
A single score across passing/failing/flaky cases, weighted by surface and risk, updated in real time. Engineering leads stop polling individual contributors for status and start defending the score itself.
